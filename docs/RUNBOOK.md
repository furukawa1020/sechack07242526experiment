# 実験運用手順

対象プロトコル: `R8-010-2x2-mock-v1`

## 1. 中止を最優先する条件

次のいずれかがあれば新しい提示を始めず、物理緊急停止、アプリの「緊急停止」、電源・空圧系の安全な停止を装置担当者の手順に従って実施します。

- 装置の異音、異臭、過熱、想定外の膨張、空気漏れ
- 参加者またはスタッフから中止の申し出
- STOPまたはDEFLATEの応答が確認できない
- USB切断、装置fault、参加者画面の切断
- 固定値、提示順、文言、時間が承認版と異なる

ソフトウェア表示より物理安全を優先します。異常セッションは再開せず、新しい研究用IDのセッションでやり直します。

## 2. 前日までの確認

- [ ] 研究責任者が固定模擬データ方式を承認している
- [ ] Googleフォームの研究説明、同意チェック、11問が承認版である
- [ ] `protocolVersion`が実施計画と一致する
- [ ] 参加者向け文言を第三者が`UI_COPY.md`と照合した
- [ ] Node.jsと依存関係をオフライン利用できる状態にした
- [ ] `npm run lint && npm run typecheck && npm test && npm run test:e2e && npm run build`が成功した
- [ ] 1366×768と1920×1080のスクリーンショットを確認した
- [ ] 参加者3〜5名相当のパイロットを完走した
- [ ] iPhoneとAndroidでGoogleフォームを確認した
- [ ] 実参加者ログがGit、クラウド同期、バックアップ対象へ入らない設定にした

## 3. 実機安全確認

装置担当者と2名で確認します。

- [ ] 実機側の最大圧力・最大膨張上限が有効
- [ ] 通信断時に実機単体で安全停止・排気できる
- [ ] 物理緊急停止が動作
- [ ] 配線とチューブに抜け、折れ、圧迫がない
- [ ] 空気漏れがない
- [ ] STOP後の安全状態とDEFLATE受理条件が確認済み
- [ ] 6秒膨張、保持、6秒収縮が承認範囲内
- [ ] 収縮完了をSTATUSまたは装置イベントで確認できる

`/device-test`を開き、接続→PING→STATUS→膨張テスト→収縮→STOPの順に確認します。本番セッション中にデバイステスト画面を操作しません。

## 4. 当日の起動

1. PCの不要なクラウド同期、通知、スリープを停止します。
2. 外部ネットワークを使用しない構成にします。Googleフォーム用端末は別経路で構いません。
3. `config/experiment.json`のSHA-256、プロトコル版、装置モード、COMポートを確認します。
4. `npm run build`後、`npm run start`で起動します。
5. `/healthz`が`ok`を返すことを確認します。
6. `/operator`と参加者用ディスプレイを開きます。
7. 表示をフルスクリーンにし、スクロールや欠けがないことを確認します。
8. 会場照明、表示距離、フォント可読性、フグの設置方向を確認します。
9. 練習用研究IDで1セッション完走後、そのログを実参加者データと分離します。

## 5. 参加者ごとの手順

1. 参加者が自身の端末でGoogleフォームの研究説明を読み、同意チェックを完了したことを確認します。
2. 氏名やメールではなく、割り当て済み研究用IDだけをスタッフ画面へ入力します。
3. 同意確認済みにチェックし、自動割付を使用します。手動選択時は偏り警告を確認します。
4. セッションを作成し、参加者画面URLを専用画面で開きます。
5. 参加者画面が接続済み、装置がreadyかつdeflated、設定検証済みであることを確認します。
6. 共通導入を表示し、参加者が読めたことを確認してから開始します。
7. 自動進行中は通常操作を行いません。残り時間、画面接続、装置状態だけを監視します。
8. サマリー表示後、参加者へGoogleフォームの11問への回答を案内します。
9. 回答完了を口頭で確認します。回答内容はアプリへ入力しません。
10. 「回答完了を確認」でセッションを完了し、次の参加者へリセットします。

## 6. 異常時

### 緊急停止

1. スタッフ画面の「緊急停止」を押します。
2. 必要なら物理緊急停止を押します。
3. STOP、DEFLATE、参加者画面の中断表示を確認します。
4. 参加者へ技術詳細を説明せず、その場で待つよう案内します。
5. 装置担当者が安全確認するまで再接続・再開しません。

### 参加者画面の更新・切断

- 通常条件ではタイマーが停止し、スタッフ画面に復旧確認が出ます。
- 画面を再接続しただけでは再開しません。表示と残り時間を確認後、「再開」を明示操作します。
- フグ動作中の切断はSTOP/DEFLATEとerrorを優先し、そのセッションを再開しません。

### 装置切断・ACKタイムアウト・fault

- アプリはSTOPと可能な場合のDEFLATEを試行します。
- 通信断ではコマンドが届かない可能性があるため、実機側の安全停止と物理状態を必ず確認します。
- セッションをerrorまたはabortedのまま保存し、続きから再開しません。

## 7. 終了処理

1. 新規セッションがないことを確認します。
2. 装置をDEFLATEし、idle/deflatedを確認します。
3. アプリを終了してから装置担当者の手順で電源を切ります。
4. JSONL件数、研究用ID、終了状態を確認し、必要なCSVを暗号化された承認済み保存先へ移します。
5. ローカルの保持・削除は研究計画の保管期間に従います。
6. 実ログをGit、チャット、issue、テストデータへ添付しません。
