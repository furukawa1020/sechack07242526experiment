# Googleフォームの外部運用境界

対象URL: `https://forms.gle/BeShY7cY5zMjunto9`

この文書名は旧版との参照互換のため維持しているが、Googleフォームは`R8-010-2x2-screen-v3`アプリのrelease/startゲートではない。

## formal productionとの分離

- production設定は`formUrl=""`
- production設定に`formAudit`を含めない
- 参加者UIとOperator UIにフォーム名、URL、リンク、QRコードを表示しない
- アプリはフォームの回答、送信、完了を取得・表示・複製・検証しない
- フォームの公開内容ハッシュ、確認者情報、照合人数を設定、manifest、ログへ保存しない
- フォーム監査の結果によりアプリが「承認済み」「未承認」と判定しない

研究スタッフはアプリ外でGoogleフォームを案内する。提示開始前に研究説明と同意を確認し、各提示直後に今見た提示について11項目へ回答できるよう案内する。

## 正しい4条件・回答フロー

1. 研究説明を読む。
2. 参加に同意する場合だけ同意チェックを行う。
3. 第1提示を見る。
4. 第1提示について11項目へ回答する。
5. 第2提示を見る。
6. 第2提示について11項目へ回答する。
7. 第3提示を見る。
8. 第3提示について11項目へ回答する。
9. 第4提示を見る。
10. 第4提示について11項目へ回答する。
11. スタッフの案内で終了する。

4条件は、クラウド×状態ラベル、この端末内×状態ラベル、この端末内×画面上のフグのふくらみ、クラウド×画面上のフグのふくらみである。参加者向けフォームにA/B/C/Dの内部記号を表示しない。

各提示後、アプリは中立な`response`で停止する。`confirm-response-checkpoint`は次の提示へ進むためのスタッフ操作であり、Googleフォームの回答内容、送信、完了をアプリが確認したことを意味しない。

## 文面の正

ユーザーから提示された研究説明を4条件版へ正規化した文面は[FORM_COPY_CORRECTED.md](FORM_COPY_CORRECTED.md)に置く。旧「3種類」、3条件だけの列挙、4提示後の一括回答、同意を別経路へ分ける説明は現行フローとして使用しない。

本システムはフォームや倫理承認資料を検証しない。フォームの運用責任と承認証跡は本システム外で管理する。
