## 解除条件

1. 通常画面と公開payloadの両方からA〜D、固定対応、旧3種類説明、旧回答手順を削除する。
2. 固定模擬データ、本人非測定、生体データ非取得、画面上のフグ、Googleフォーム回答は明示送信時にGoogleへ送信・保存されること、実験アプリは固定模擬データを外部へ送信・保存しないことを、それぞれ承認済み文言で明記する。
3. 質問文が空の項目、必須の年齢自由記述、氏名・メール等の個人情報項目、研究用ID以外の短文・段落自由記述、ファイルアップロードをなくす。
4. 第1〜第4提示を評価する承認済み11問、7件法、任意回答が全経路で一致することを確認する。
5. Google Forms管理画面で、メール収集、ログイン要求、組織制限、回答控えの特定可能な送信を無効にする。
6. 提示開始前の同意記録方法、研究用IDで回答とローカル提示順を連結する方法、撤回・除外・削除方法を研究責任者が承認する。フォームの研究用ID欄は固定ラベル`研究用ID`の必須短文入力1件、`^SH26-[0-9]{3}$`との完全一致validationとし、提示順、順序コード、A〜Dを入力させない。
7. Googleフォームの回答経路を未ログイン状態、iPhone、Androidで送信完了まで二名が独立に完走する。提示前同意を別経路で記録する場合は、その承認済み同意経路も別途二名で確認する。
8. `npm run audit:form`の全機械所見がPASSすることを確認する。
9. `formAudit`へ、同じprotocolVersion、URL、7日以内の日付、修正後の安定公開payload SHA-256、`twoPersonVerified=true`を記録する。
10. 配布URLにprefill値、未知のquery、fragmentがないことを確認し、リリース直前に`npm run verify:form-release -- --config config/experiment.production.json`を実行して同じ公開payloadであることを再確認する。
