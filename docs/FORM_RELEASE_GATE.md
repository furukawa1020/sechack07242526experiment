## 現在のブロッカー

- 公開payloadに内部コードA〜Dと固定対応が残っている
- フォームタイトルが提示後評価専用の固定タイトルではなく、提示後フォームを同意取得経路と誤認させる`研究説明・参加同意・アンケート`のままである
- 「3種類」と「4種類」の説明が併存している
- 「各提示直後」と「4提示後」の回答手順が併存している
- 固定模擬データ、本人非測定、生体データ非取得、画面上のフグ、Googleフォーム回答の送信先、実験アプリによる固定模擬データの外部非送信・非保存というscreen版の必須6点の説明がない
- 年齢説明付近に質問文が空の必須入力項目があり、公開監査だけでは入力意図・収集内容を確定できない
- 研究用IDと提示順コードの入力を求める説明に対し、意図した入力欄の構造が一致していない
- 研究用ID欄のラベルが`研究用ID（必須）`で固定ラベル`研究用ID`と一致せず、`^SH26-[0-9]{3}$`との完全一致validationがない
- 参加者へ提示順コードの入力を求める説明が残っている
- 氏名・メール等の禁止された個人情報を入力・収集する説明または回答項目を1件検出している
- 厳密な研究用ID欄以外の短文・段落自由記述を3件検出している
- 回答項目が「厳密な研究用ID欄1件＋評価グリッド11件」の計12件だけという許可リストに一致しない
- メール収集、ログイン要求、組織制限等の管理画面設定を所有者が未確認
- 提示開始前の同意を、この事後評価フォームとは別のどの承認済み経路へ記録するかが未確定

## 解除条件

1. フォームタイトルを`身体状態の外化デバイスがユーザの心理状態に及ぼす影響の評価｜提示後アンケート`へ完全一致させ、通常画面と公開payloadの両方からA〜D、固定対応、旧3種類説明、旧回答手順を削除する。
2. 固定模擬データ、本人非測定、生体データ非取得、画面上のフグ、Googleフォーム回答は明示送信時にGoogleへ送信・保存されること、実験アプリは固定模擬データを外部へ送信・保存しないことを、それぞれ承認済み文言で明記する。
3. 質問文が空の項目、年齢・属性・同意、氏名・メール等の個人情報項目、研究用ID以外の短文・段落自由記述、ファイルアップロードを含む、許可外の回答項目をすべてなくす。
4. 回答項目全体を、固定ラベル・必須・完全一致validationを備えた研究用ID欄1件と、第1〜第4提示を承認済み7件法・任意回答で評価する11グリッドの計12件だけにする。追加の選択式、プルダウン、チェックボックス、尺度、日付、時刻、未知形式も置かない。
5. Google Forms管理画面で、メール収集、ログイン要求、組織制限、回答控えの特定可能な送信を無効にする。
6. 提示開始前の研究説明と同意は、この事後評価フォームとは別の承認済み経路で提示前に提供・記録する。研究用IDで回答とローカル提示順を連結する方法、撤回・除外・削除方法と併せて研究責任者が承認する。フォームへ提示順、順序コード、A〜Dを入力させない。
7. Googleフォームの回答経路を未ログイン状態、iPhone、Androidで送信完了まで二名が独立に完走する。別の承認済み提示前同意経路も二名が独立に確認する。同意チェックを評価フォームから削除しただけでは解除条件を満たさない。
8. `npm run audit:form`の全機械所見がPASSすることを確認する。
9. `formAudit`へ、同じprotocolVersion、URL、7日以内の日付、修正後の安定公開payload SHA-256、`twoPersonVerified=true`を記録する。
10. 配布URLにprefill値、未知のquery、fragmentがないことを確認し、リリース直前に`npm run verify:form-release -- --config config/experiment.production.json`を実行して同じ公開payloadであることを再確認する。
