対象プロトコル: `R8-010-2x2-mock-v2`

監査記録日: 2026-07-20

対象URL: `https://forms.gle/BeShY7cY5zMjunto9`

判定: **NO-GO**

この記録は公開フォームの読取り専用監査結果である。フォームへの回答送信、編集、設定変更、管理画面へのアクセスは行っていない。フォーム所有者が内容を修正し、再監査と二名照合を完了するまで、本番参加者へ案内してはならない。

## 1. 監査範囲

公開URLへのHTTP応答と、公開応答に含まれる文字列を機械的に確認した。Google Formsの編集者画面、回答保存先、アカウント制限、メールアドレス収集設定等の管理者専用設定は監査対象外である。

「公開payload」は、参加者がアクセスできる公開応答に含まれるデータを指す。通常画面で常時表示される文字列だけを意味しない。

## 2. 機械的に確認できた事実

| 項目           | 監査結果                                                                            |
| -------------- | ----------------------------------------------------------------------------------- |
| HTTP応答       | HTTP 200を返した                                                                    |
| 研究タイトル   | 監査対象として想定した研究タイトルと一致した                                        |
| 内部条件       | 公開payloadに、参加者へ見せない内部コードA〜Dと条件の固定対応が記載されていた       |
| 提示種類数     | 3種類と説明する文面と、4種類と説明する文面が同じ公開payload内に併存していた         |
| 回答タイミング | 各提示直後に11項目へ回答する説明と、4提示終了後にまとめて回答する説明が併存していた |
| 評価項目数     | 公開フォーム構造の再解析で11評価項目を確認した                                      |

HTTP 200と研究タイトルの一致は、URLが対象フォームを返したことだけを示す。研究説明、同意、質問数、回答経路、個人情報非収集が承認済みであることは示さない。

## 3. 機械監査では判定できない事項

- 11評価項目の存在は確認したが、文言・尺度・必須設定が承認版と一言一句一致するかは二名による実回答経路の照合が必要である。
- メールアドレス収集が無効かどうかは判定していない。表示文面にメール入力欄が見当たらないことだけでは、Google Formsのメール収集設定が無効である証拠にならない。
- 氏名等の個人情報を必須にする条件分岐、Googleアカウントへのログイン要求、回答控え送信、回答後の編集可否は管理画面と実際の回答経路を人手で確認する必要がある。

## 4. リスク評価

次は監査事実から導く運用上の判定であり、機械抽出された事実そのものではない。

- 内部コードと固定対応を公開payloadへ含めることは、参加者へA/B/C/Dを見せない研究条件に反し、仮説推測や回答バイアスを招くおそれがある。
- 3種類と4種類の併存、および回答タイミングの併存は、参加者への説明と実際の実験手順を一致させられない。
- 11評価項目は確認できたが、内部コード公開、提示数、回答タイミングの不整合が残るため、質問数の一致だけでは本番利用を承認できない。
- メール収集設定が未確認の状態では、個人情報を収集しない要件を満たしたと判断できない。

以上のため、フォーム内容は現状のまま本番利用できない。

## 5. NO-GO解除条件

フォーム所有者が承認済み研究計画に基づいて修正した後、次を二名が独立に確認する。

- [ ] 公開画面と公開payloadのどちらにも、内部コードA/B/C/Dおよび固定対応が含まれない
- [ ] 提示数の説明が4提示に統一され、3種類という旧説明が残っていない
- [ ] 回答手順が、4提示とサマリーの終了後にGoogleフォームへ戻って回答する承認済みフローへ統一されている
- [ ] 第1提示から第4提示までを評価する11問が、実際の回答経路で過不足なく表示・回答できる
- [ ] 研究説明、同意チェック、11問および送信完了画面が承認版と一言一句一致する
- [ ] Google Forms管理画面でメールアドレス収集が無効であることを確認する
- [ ] ログイン要求、氏名、メールアドレスその他の個人情報収集がないことを、未ログイン状態の実回答経路でも確認する
- [ ] QRとURLが同じ対象フォームを開く
- [ ] iPhoneとAndroidの両方で、分岐を含む全回答経路を完走する
- [ ] 修正後の公開payloadを再監査し、本書の未解決所見が0件になったことを記録する
- [ ] 確認者2名の氏名、日時、確認したフォーム版または承認済み証跡を研究計画で定めた場所へ記録する

参加者向け説明または回答手順を変更する場合は、研究責任者が研究計画との整合を確認し、必要に応じて`protocolVersion`と`docs/PROTOCOL_CHANGELOG.md`を更新する。本書のチェックだけを先に完了扱いにしてはならない。
