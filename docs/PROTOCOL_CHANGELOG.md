# プロトコル変更履歴

## R8-010-2x2-screen-v1 — 2026-07-21

- 正式MVPの入力を、参加者本人を測定したものではない同一の固定模擬データとして明記し、心拍その他の生体データを取得しないことを参加者向け固定文言へ追加。
- フグ条件の刺激を物理フグから参加者画面内のフグへ変更し、右パネル文言を「状態は画面上のフグの／ふくらみで表されています」に固定。
- 正式実施用の`device.mode=screen`と`ScreenPufferDevice`を、障害注入可能な開発・テスト・模擬リハーサル専用`mock`から分離。
- C/Dでは、`result`開始から6,000msで同じ画面上フグを膨張させ、結果提示終了まで保持し、`reset`開始から6,000msで収縮させる。命令列、描画、目標レベル、開始時刻、速度および保持時間を完全に同一とする。
- 画面上フグの進捗は継続接続中のサーバ時刻に同期する。通常フェーズの再読み込みはOperator確認まで停止し、`result`または`reset`中の切断・再読み込みは刺激欠損としてSTOP、DEFLATE、errorとし再開しない。
- 正式画面版はUSB機器を必要とせず、フグ制御のための外部通信を行わない。`serial`による物理フグは将来の別プロトコルへ分離。
- 参加者向け共通文言を「固定模擬データ」「本人の測定ではない」「生体データ非取得」「画面上だけのフグ」に統一。
- A=cloud+label、B=local+label、C=local+puffer、D=cloud+pufferの内部対応、提示順ABDC / BCAD / CDBA / DACB、固定値72 / 高ストレス / 0.60、8,000 / 3,000 / 15,000 / 7,000msのフェーズ時間は変更していない。
- 運用上のフェイルクローズ強化として、フォーム監査とは別に研究計画、倫理判断、提示前同意、データ管理、screenパイロット、独立二名照合を非個人識別の`goEvidence`へ結び付け、manifest v4で設定・証跡・appVersion・production設定だけを除外したGit HEAD追跡tree SHA-256・source commitを封印した。
- 研究用ID単位のread-only Previewと保持期限レポートを追加した。アプリ内の分析除外・削除は安全上常に拒否し、正式リリースへ変更用CLIを同梱しない。変更は研究責任者が事前承認した外部手順へ引き渡し、Googleフォーム回答は同じ研究用IDをフォーム管理者が別途手動照合する。
- セッション作成時に研究用IDを排他的・永続的に予約するregistryを追加した。ログの移動・削除後もregistryを保持し、IDを再割付しない。
- Googleフォーム公開payload監査へ、研究用ID欄の固定ラベル・必須設定・形式validationと、提示順・内部コード入力の不存在を追加した。
- 初回GO前のscreenパイロット循環を解消するため、研究チームの非参加者専用`screen-pilot`経路を追加した。正式固定値・時間・順序とScreenPufferDeviceを使う一方、`PILOT-xxx`、空フォーム、loopback、隔離ログ、非参加者表示を強制し、正式リリースへ同梱しない。
- Operator WebSocketへ1秒challenge/nonce応答・5秒の往復leaseを追加し、silent LAN断、上り・下りのhalf-open、ブラウザ停止を検出する。確認済みleaseがなければ提示準備・開始・再開を拒否し、進行中の最後の有効lease喪失ではSTOP・DEFLATE・`OPERATOR_CONNECTION_LOST`とする。複数接続の1つが生存中は継続する。
- Googleフォーム監査は、内部コードと伝え方だけの対応および「3つの提示」「3提示」等の旧表現も拒否するよう強化した。
- これらは本番開始条件、非参加者確認、監視安全性、データ管理の実装強化であり、正式参加者が見る文言、条件、表示、提示順、タイミング、固定値、画面上フグ動作は変更していないため、protocolVersionは維持した。

物理フグから画面上フグへの変更は参加者が見る研究刺激の変更である。実参加者へ使用する前に、研究責任者の承認を得て、所属機関で必要とされる倫理審査・研究計画の変更手続きを完了する。

## R8-010-2x2-mock-v3 — 2026-07-21

- 正式参加者画面に残っていた左上寄せと片寄った空白を解消し、主要内容を利用可能な表示領域の中央で均衡する配置へ変更。
- handling画面の同心円・点による装飾を削除し、研究表示に不要なAI風の視覚モチーフを撤去。
- 共通導入、handling、processing、結果表示、サマリーについて、1366×768と1920×1080で中央配置と画面内収まりを自動確認。
- 実機なしの専用模擬リハーサルに限り、研究参加用ではないこと、回答送信なし、実機なしを常設し、サマリーのGoogleフォーム誘導を模擬終了案内へ置換。
- 模擬リハーサル表示を明示的なloopback専用起動モードへ結び付け、通常の開発用Mock設定をリハーサルと誤表示しないよう修正。
- 再利用用Codexパッケージの実験仕様と固定文言を同じv3要件へ同期。
- 正式実験の条件対応、参加者向け固定文言、提示順、タイミング、固定値、A/Bの右表示、C/Dの右表示と装置命令、外部通信方針は変更していない。

参加者が見る刺激の配置と顕著性が変わるため、実験の再現性を優先してプロトコルバージョンを更新した。
