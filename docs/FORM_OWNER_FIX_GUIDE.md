# Googleフォーム所有者向け整合ガイド

対象URL: `https://forms.gle/BeShY7cY5zMjunto9`

> このGoogleフォームは`R8-010-2x2-screen-v3`のformal productionアプリ外で運用する。フォームの内容、回答、完了状態、URLをアプリ、設定、manifest、実験ログへ取得・複製しない。本書の確認結果やフォーム監査はアプリのrelease/startハードゲートではない。

## 正とする実験説明

- 条件数は4条件
- A = クラウド × 状態ラベル
- B = この端末内 × 状態ラベル
- C = この端末内 × 画面上のフグのふくらみ
- D = クラウド × 画面上のフグのふくらみ
- 参加者向けにはA/B/C/Dの内部記号を表示しない
- 全条件で同じ固定模擬データを使用し、参加者本人を測定しない
- 心拍、体温その他の生体データを取得しない
- 物理フグやUSB機器ではなく、画面上のフグを使用する
- クラウド条件は比較用シナリオであり、実験アプリは身体データを外部送信・保存しない
- 各提示の直後に、今見た提示について11項目へ回答する
- 4提示後にまとめて初めて回答する手順ではない
- 研究説明、参加同意、アンケート回答は同じGoogleフォームにまとめる
- 年齢区分は任意

フォームへ貼り付ける研究説明は[4条件版の正規化済みフォーム文面](FORM_COPY_CORRECTED.md)を参照する。ユーザーから提示された研究説明の内容を保ち、旧「3種類」だけを「4種類」へ修正し、第4のクラウド×画面上フグ条件を明記する。

## 参加前の同意

1. 研究スタッフが対面で案内する。
2. 参加者がGoogleフォームの研究説明を最後まで読む。
3. 参加者が「説明内容を理解し、研究への参加に同意します」にチェックする。
4. 同意しない場合は実験を開始しない。
5. Operatorは対象参加者について同意完了をセッション内で確認する。

Operator確認は同意記録そのものではない。氏名、メールアドレス、Googleアカウント、回答内容をアプリへ入力・取得・保存しない。

## 各提示直後の11項目

アプリは各提示の`reset`後に`response`で停止し、参加者画面には次の2文だけを表示する。

```text
第{n}提示は終了しました
研究スタッフの案内をお待ちください。
```

研究スタッフはアプリ外で、参加者が今見た提示に対応する11項目へ回答できるよう案内する。アプリへフォーム名、URL、QRコード、回答内容、送信状況、完了状態を入力・表示しない。Operatorの`confirm-response-checkpoint`は待機表示と次の進行準備を確認する操作であり、回答内容・送信・完了をアプリが検証したことを意味しない。

## 収集設定

ユーザーが提示した研究説明に合わせ、フォーム所有者は次を確認する。

- 氏名、メールアドレス、住所、電話番号、学籍番号を収集しない
- Google Formsの「メールアドレスを収集する」を使用しない
- ファイルアップロード、顔画像、動画、音声、生体データを収集しない
- 研究用ID、提示順コード、任意の年齢区分、アンケート回答、回答日時だけを研究計画に沿って扱う
- 回答したくない評価項目を空欄にできる
- 同意しない参加者を実験回答セクションへ進めない
- 参加者に内部条件コードA/B/C/Dを回答させない

フォームの共有権限、回答保存先、撤回、削除は研究責任者が本システム外で管理する。
