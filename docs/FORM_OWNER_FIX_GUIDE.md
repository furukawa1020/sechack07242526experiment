# Googleフォーム所有者向け修正ガイド

対象URL: `https://forms.gle/BeShY7cY5zMjunto9`

このガイドは、[Googleフォーム公開内容監査](FORM_AUDIT.md)で確認された不整合をフォーム所有者が修正するための作業票である。研究説明、同意文、質問文、尺度、必須設定を新しく承認する文書ではない。修正後も、二名による再監査と研究責任者の承認が完了するまで本番参加者へ案内せず、判定をNO-GOのままとする。

## 1. 先に削除する内容

フォームの通常画面だけでなく、説明、選択肢、分岐先、送信後メッセージ、非表示または未使用のセクションを含む全項目から次を削除する。

- 内部コード`A`、`B`、`C`、`D`と、各コードに対応する処理場所・提示方法の説明
- 「3種類」「3つの提示」など、提示数を3とする旧説明
- 各提示の直後に11項目へ回答するよう求める旧説明
- 4提示終了後にまとめて回答する手順と矛盾する案内
- 仮説、期待する結果、条件の良し悪しを推測させる説明
- 氏名、メールアドレス、学籍番号、住所、電話番号その他の個人情報を入力させる項目
- 年齢説明付近にある質問文が空の必須入力項目（公開監査だけでは入力意図・収集内容を確定できない）
- 研究用ID欄以外の短文・段落による自由記述項目

削除対象を別の説明欄へ移動しただけでは解消にならない。公開payloadに旧文面が残らないよう、不要なセクションと分岐も削除する。

## 2. 実験手順の貼り替え案

研究責任者が承認済み研究計画および同意文と照合した後、手順説明を次へ統一する。承認前に公開しない。

```text
この実験では、同じ固定模擬データを4つの方法で提示します。
表示される値は、あなた自身を測定したものではありません。
この実験では、心拍などの生体データを取得しません。

変わるのは、「どこで処理する設定か」と「状態ラベルまたは画面上のフグのふくらみでどう伝えるか」です。

アプリで4つの提示をすべて見終え、サマリーが表示された後、このフォームへ戻ってください。

第1提示から第4提示までを、11問でそれぞれ評価してください。
回答は、4つの提示がすべて終了してから行ってください。
```

比較用シナリオの説明をフォームにも置く場合は、研究責任者の承認後、次の固定文言を使用する。

```text
これから表示されるデータの取扱いは、比較のためのシナリオです。
この実験用Webアプリから、固定模擬身体データを外部へ送信・保存することはありません。
アンケート回答は、あなたがGoogleフォームで送信したときにGoogleへ送信・保存されます。
```

参加者向けには第1提示〜第4提示とだけ記載し、内部コードや固定対応を記載しない。

## 3. 研究説明と同意

- 研究説明と同意文は研究責任者が承認した版を一言一句そのまま使用する。
- このガイドを根拠に同意文を新規作成、要約、言い換えしない。
- 同意チェックの文言、必須設定、撤回・中止・問い合わせ案内は承認済み版と照合する。
- 参加者は提示開始前に承認済みの方法で研究説明を読み、同意を記録する。スタッフは同意文や回答内容をアプリへ複製せず、承認済み手順の完了だけを確認してからアプリの同意確認欄をチェックする。
- 同じGoogleフォームを提示後に送信する方式では、同意の記録時刻が提示後になる。研究責任者は、これでよいか、同意だけを別フォームまたは書面で提示前に記録するかを開始前に決定し、承認済み手順へ固定する。
- 説明または同意手順を変更する場合は、研究責任者が研究計画との整合とプロトコル変更の要否を判断する。

## 4. 研究用IDと提示順の連結

- Googleフォームには、ラベルを一言一句`研究用ID`とした必須の短文入力を1件だけ置く。ラベルへ`（必須）`等を足さず、Google Formsの必須設定で示す。
- 研究用ID欄には正規表現`^SH26-[0-9]{3}$`との完全一致validationを設定し、スタッフから割り当てられた研究用IDだけを入力する。氏名、メールアドレス、学籍番号は入力しない。
- 同じ研究用IDをスタッフ画面へ入力し、ローカル監査ログ内で研究用IDと提示順を連結する。
- 参加者向けフォームへ提示順、順序コード、内部コードA/B/C/D、固定対応を記載せず、入力・選択も求めない。
- 回答データとローカルログを研究用IDで結合する手順、アクセス権、保存期間を研究責任者が承認済み計画へ固定する。

## 5. Google Formsの収集設定

フォーム所有者が管理画面で、少なくとも次の結果になるよう設定する。Google Formsの画面上の名称が変わっている場合も、機能上の結果で確認する。

- メールアドレスを収集しない
- Googleアカウントへのログインを要求しない
- 組織内ユーザーだけに制限しない
- 回答者を特定する回答控えの自動送信を使用しない
- ログインを必要とする「回答を1回に制限」を使用しない
- ファイルアップロード質問を使用しない
- タイトルや質問文が空の回答入力項目を残さない
- 現在検出されている年齢付近の無題必須入力は削除する。年齢が承認済み計画で必要な場合だけ、任意の粗い年代区分と「回答しない」へ置き換え、質問数とデータ管理を承認版へ反映する
- 氏名、メールアドレスその他の個人情報を質問しない
- `研究用ID`以外に短文・段落の自由記述を置かない
- 第1提示〜第4提示を評価する承認済み11問だけが、意図した順序・尺度で表示される
- 各質問の必須・任意設定を承認済み版と照合し、回答したくない項目を空欄にできるという参加者向け案内と矛盾させない

回答保存先、共同編集者、共有権限、保持期間についても研究計画で承認された範囲だけを使用する。

## 6. 公開前の所有者確認

- [ ] 通常画面と公開payloadの両方から内部コードA〜Dと固定対応が消えている
- [ ] 提示数が4へ統一され、「3種類」等の旧文面がない
- [ ] 回答タイミングが4提示とサマリーの終了後へ統一されている
- [ ] 承認済みの研究説明、同意、11問、尺度、必須設定と一言一句照合した
- [ ] 管理画面でメール収集、ログイン要求、組織制限、個人情報質問がない
- [ ] タイトルや質問文が空の回答入力項目がない
- [ ] 必須の年齢自由記述がなく、年齢を扱う場合は承認済みの任意・粗い区分と「回答しない」だけである
- [ ] 本人を測定していない固定模擬データであり、生体データを取得しないことが明記されている
- [ ] フグは画面上の表現であり、物理装置を使用しないことが明記されている
- [ ] 実験用Webアプリから固定模擬身体データを外部へ送信・保存しないことが明記されている
- [ ] アンケート回答は参加者がGoogleフォームで送信したときにGoogleへ送信・保存されることが明記され、実験アプリの非送信・非保存と区別されている
- [ ] 提示開始前の同意確認・記録方法を研究責任者が承認し、スタッフ手順と一致している
- [ ] 研究用IDで回答とローカル提示順を連結する承認済み手順があり、フォームに提示順や内部コードを含めていない
- [ ] `研究用ID`という必須短文入力が1件だけあり、`^SH26-[0-9]{3}$`との完全一致validationがある
- [ ] 配布URLに`entry.*`等の事前入力値、未知のquery、fragmentがない。queryを付ける場合は`usp=send_form`だけである
- [ ] 未ログインの新しいブラウザセッションで全回答経路を確認した
- [ ] QRコードとURLが同じフォームを開く
- [ ] iPhoneとAndroidで全回答経路を確認した

## 7. 修正後の再監査

所有者自身の確認だけでGOにしない。修正後の公開URLについて、二名が独立に[FORM_AUDIT.mdの解除条件](FORM_AUDIT.md#5-no-go解除条件)を確認し、同書の再監査記録へ確認者、日時、フォーム版または承認済み証跡、研究責任者の承認を記録する。

未確認項目、旧文面、設定不明、承認待ちが1件でも残る場合はNO-GOを継続する。
