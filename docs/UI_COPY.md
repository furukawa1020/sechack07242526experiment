# UI_COPY.md — 参加者向け固定文言

このファイルを参加者向け文言の単一の正とする。実験実施版では、文言をスタッフ画面から編集できないようにする。

## 1. 共通導入

### 見出し

```text
同じ身体データを、4つの方法で提示します
```

### 本文

```text
変わるのは、「どこで処理するか」と「どう伝えるか」です。

あなたは、少し本調子ではないまま作業を続けている場面を想定してください。

どの方法が正しいかを選ぶ課題ではありません。
それぞれを見たときに、どう感じたかを覚えておいてください。
```

### シナリオ説明

```text
これから表示されるデータの取扱いは、比較のためのシナリオです。
この実験で、実際の身体データをクラウドへ送信・保存することはありません。
```

### 開始待ち

```text
研究スタッフの案内をお待ちください
```

## 13. 模擬リハーサル専用表示

次の文言は、専用ランチャーから明示的な`rehearsal`起動モードで開始し、かつ`device.mode=mock`、空のフォームURL、loopback限定をすべて満たす模擬リハーサルでだけ表示する。条件値が同じでも通常の開発起動では表示しない。正式実験の刺激や参加者向け文言には含めない。

### 常設表示

```text
模擬リハーサル
```

```text
研究参加用ではありません・回答送信なし・実機なし
```

### 模擬サマリー

```text
4つの模擬提示を確認しました。
Googleフォームへの回答や送信は行いません。
```

### 模擬完了

```text
模擬リハーサルを終了しました
```

```text
この画面は研究参加の完了を示すものではありません。
```
