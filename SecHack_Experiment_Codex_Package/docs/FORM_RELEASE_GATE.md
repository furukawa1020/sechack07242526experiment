2026-07-21時点の判定は**NO-GO**である。フォーム所有者による修正、読取り専用の機械再監査、二名の独立照合、研究責任者の承認が完了するまで、実参加者へURLやQRを案内せず、`formAudit.status`を`GO`へ変更しない。

## 現在のブロッカー

- 公開payloadに内部コードA〜Dと固定対応が残っている
- 「3種類」と「4種類」の説明が併存している
- 「各提示直後」と「4提示後」の回答手順が併存している
- 固定模擬データ、本人非測定、生体データ非取得、画面上のフグ、アプリ非送信とGoogleフォーム回答送信の区別というscreen版の必須説明がない
- 年齢説明付近に質問文が空の必須入力項目があり、公開監査だけでは入力意図・収集内容を確定できない
- 研究用IDと提示順コードの入力を求める説明に対し、意図した入力欄の構造が一致していない
- メール収集、ログイン要求、組織制限等の管理画面設定を所有者が未確認
- 提示開始前の同意をいつ・どこへ記録するかが未確定

## 解除条件

1. 通常画面と公開payloadの両方からA〜D、固定対応、旧3種類説明、旧回答手順を削除する。
2. 固定模擬データ、本人非測定、生体データ非取得、画面上のフグ、アプリからの実クラウド送信なしとGoogleフォーム回答は明示送信時にGoogleへ送信・保存される区別を承認済み文言で明記する。
3. 質問文が空の項目、必須の年齢自由記述、氏名・メール等の個人情報項目、ファイルアップロードをなくす。
4. 第1〜第4提示を評価する承認済み11問、7件法、任意回答が全経路で一致することを確認する。
5. Google Forms管理画面で、メール収集、ログイン要求、組織制限、回答控えの特定可能な送信を無効にする。
6. 提示開始前の同意記録方法、研究用IDで回答とローカル提示順を連結する方法、撤回・除外・削除方法を研究責任者が承認する。フォームへ提示順やA〜Dを入力させない。
7. Googleフォームの回答経路を未ログイン状態、iPhone、Androidで送信完了まで二名が独立に完走する。提示前同意を別経路で記録する場合は、その承認済み同意経路も別途二名で確認する。
8. `npm run audit:form`の全機械所見がPASSすることを確認する。
9. `formAudit`へ、同じprotocolVersion、URL、7日以内の日付、修正後の安定公開payload SHA-256、`twoPersonVerified=true`を記録する。
10. リリース直前に`npm run verify:form-release -- --config config/experiment.production.json`を実行し、同じ公開payloadであることを再確認する。

HTTP 200、タイトル一致、11問の存在だけではGOにならない。既知NO-GOのSHA-256を転記したまま、フラグだけを変更しても本番ゲートは通してはならない。
