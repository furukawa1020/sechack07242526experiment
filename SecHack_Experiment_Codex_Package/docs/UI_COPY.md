# UI_COPY.md — 参加者向け固定文言

このファイルを参加者向け文言の単一の正とする。実験実施版では、文言をスタッフ画面から編集できないようにする。

## 1. 共通導入

### 見出し

```text
同じ固定模擬データを、4つの方法で提示します
```

### 本文

```text
変わるのは、「どこで処理するか」と「どう伝えるか」です。

あなたは、少し本調子ではないまま作業を続けている場面を想定してください。

どの方法が正しいかを選ぶ課題ではありません。
それぞれを見たときに、どう感じたかを覚えておいてください。
```

### シナリオ説明

```text
これから表示される値は、比較のために用意した同じ固定模擬データです。
表示される値は、あなた自身を測定したものではありません。
この実験では、心拍その他の生体データを取得しません。

これから表示されるデータの取扱いは、比較のためのシナリオです。
この実験用Webアプリから、固定模擬身体データを外部へ送信・保存することはありません。

画面上のフグは表示だけの表現で、USB機器や実機は接続・動作していません。
```

### 開始待ち

```text
研究スタッフの案内をお待ちください
```

## 9. 提示後のスタッフ確認待ち

各提示のリセット終了後、サーバが次の提示へ自動進行せず、研究スタッフの確認を待つ間に表示する。

### 参加者画面の見出し

```text
第{n}提示は終了しました
```

### 参加者画面の本文

```text
研究スタッフの案内をお待ちください。
```

この画面では、直前の条件刺激、内部条件コード、外部回答に関する名称・URL・QRコード・回答状況、操作ボタンを表示しない。

### スタッフ画面

```text
提示後のスタッフ確認
```

```text
参加者画面が待機表示になっていることを確認してください。
```

```text
待機表示を確認して次へ
```

この操作は各提示後の中立な進行確認であり、4提示後の最終スタッフ引継ぎ確認とは分ける。外部回答の完了確認として扱わない。

## 14. 非参加者用の事前確認表示

次の文言は、非参加者専用の`development`、`rehearsal`、`screen-pilot`、`test`でだけ表示する。各モードはloopback限定、外部runtime通信なし、外部回答の取得・表示・送信なし、専用ID、隔離ログを必須とする。`development`と`rehearsal`は`device.mode=mock`、`screen-pilot`は`device.mode=screen`と正式時間・固定値・順序、`test`は`mock`または`screen`だけを許可する。正式productionでは必ず非表示にし、正式実験の刺激や参加者向け文言へ含めない。

### 常設表示

```text
非参加者用の事前確認
```

```text
研究参加用ではありません・外部回答送信なし
```

### 事前確認サマリー

```text

外部回答の送信は行いません。
```

### 事前確認完了

```text

```

